## Amazon DynamoDB

- It is a fully managed **NoSQL database** service.
- It provides **fast performance**, **automatic scaling**, and **high availability** without managing database servers.

DynamoDB stores data as **Items** inside **Tables**.

---

---

## Basic Terminology

### Table

A **Table** stores data.

Example:

```text id="q4t2zo"
Users
```

---

### Item

An **Item** is a single record in a table.

Example:

| UserID | Name  | City |
| ------ | ----- | ---- |
| 101    | Rahul | Pune |

---

### Attribute

An **Attribute** is a column (field) in an item.

Example:

* UserID
* Name
* City

---

## Primary Key

Every table requires a **Primary Key**.

Types:

* **Partition Key** (Simple Primary Key)
* **Partition Key + Sort Key** (Composite Primary Key)

The primary key uniquely identifies each item.

---

## DynamoDB vs RDS

| DynamoDB                         | RDS                                 |
| -------------------------------- | ----------------------------------- |
| NoSQL database                   | Relational database (SQL)           |
| Tables, Items, Attributes        | Tables, Rows, Columns               |
| Flexible schema                  | Fixed schema                        |
| Automatic scaling                | Manual or configured scaling        |
| Best for high-speed applications | Best for structured relational data |

---

## Key Points

* DynamoDB is a **NoSQL** database.
* Data is stored in **Tables**.
* Each record is an **Item**.
* Each field is an **Attribute**.
* Automatically scales based on workload.
* Provides high availability and low latency.
