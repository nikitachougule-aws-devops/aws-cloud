## Amazon Machine Image (AMI)

- It is a pre-configured template used to launch an EC2 instance.
- AMI as a **blueprint** for creating an EC2 instance.

It contains everything needed to start a virtual server, including:

* Operating System
* Application software (optional)
* Configuration settings
* Storage information

---

## Why Use an AMI?

* Launch EC2 instances quickly
* Create identical servers
* Save time on server setup
* Back up server configurations

---

## AMI Components

An AMI includes:

* Operating System (Linux or Windows)
* Root Volume (EBS Snapshot)
* Launch Permissions
* Block Device Mapping (Storage configuration)

---

## Types of AMIs

### 1. AWS Provided AMIs

- Created and maintained by AWS.
- Examples: Amazon Linux, Ubuntu, Windows Server

---

### 2. Custom AMIs

- Created by users from their own EC2 instances.
- Used when you want multiple servers with the same configuration.

---

### 3. AWS Marketplace AMIs

- Provided by third-party vendors.
- Examples: WordPress, MongoDB, Jenkins, Red Hat Enterprise Linux
- Some Marketplace AMIs are free, while others require additional charges.

---

## How AMI Works

```text
AMI
 │
 ├── Operating System
 ├── Software
 ├── Configuration
 └── Storage Snapshot
        │
        ▼
Launch EC2 Instance
```

---

## Create a Custom AMI

1. Launch an EC2 instance.
2. Install required software.
3. Configure the server.
4. Create an AMI from the instance.
5. Launch new EC2 instances using the AMI.

- This ensures every new instance has the same configuration.

---

## AMI vs EC2 Instance

| AMI                      | EC2 Instance           |
| ------------------------ | ---------------------- |
| Template                 | Running virtual server |
| Used to launch instances | Created from an AMI    |
| Cannot process requests  | Runs applications      |

---

## Example

Suppose you've configured a web server with:

* Amazon Linux
* Nginx
* Docker
* Git

Instead of repeating the setup for every new server, create a **Custom AMI**.

Any EC2 instance launched from this AMI will already have the same software and configuration.


---

## Key Points

* AMI stands for **Amazon Machine Image**.
* An AMI is a template used to launch EC2 instances.
* It contains the operating system, software, and configuration.
* AWS provides public AMIs, and users can create custom AMIs.
* One AMI can be used to launch multiple EC2 instances.
