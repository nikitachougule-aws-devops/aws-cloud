## Amazon EC2 Auto Scaling

- It automatically adds or removes EC2 instances based on your application's demand.
- It helps maintain application performance while reducing costs.
1. More traffic → Add instances
2. Less traffic → Remove instances

---

## Why Use Auto Scaling?

* Handle traffic spikes automatically
* Improve application availability
* Reduce infrastructure costs
* Maintain application performance

---

## How Auto Scaling Works

```text
High Traffic
     │
     ▼
Launch More EC2 Instances
     │
     ▼
Application Handles More Users
     │
     ▼
Traffic Decreases
     │
     ▼
Terminate Extra EC2 Instances
```

---

## Auto Scaling Group (ASG)

It is a group of EC2 instances managed together.

You define:

* Minimum instances
* Desired instances
* Maximum instances

Example:

| Setting | Value |
| ------- | ----: |
| Minimum |     2 |
| Desired |     2 |
| Maximum |     5 |

AWS automatically keeps the desired number of instances running.

---

## Scaling Types

### 1. Dynamic Scaling

- Automatically scales based on metrics such as CPU utilization.
- Example: CPU > 70% → Add 1 EC2 instance, CPU < 30% → Remove 1 EC2 instance

---

### 2. Scheduled Scaling

- Scales at a specific date and time.
- Example: Increase instances every day at **9:00 AM** before business hours.

---

### 3. Manual Scaling

- You manually change the number of EC2 instances.

---

## Key Points

* Auto Scaling automatically adjusts the number of EC2 instances.
* An Auto Scaling Group (ASG) manages multiple EC2 instances.
* Supports Dynamic, Scheduled, and Manual scaling.
* Helps improve availability and reduce costs.
