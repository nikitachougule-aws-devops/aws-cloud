## Security Groups

- It is a virtual firewall that controls **inbound** and **outbound** traffic for AWS resources such as EC2 instances.
- It allows only the traffic that you explicitly permit.

---

## How Security Groups Work

```text
Internet
    │
    ▼
Security Group
    │
    ▼
EC2 Instance
```

The Security Group checks incoming and outgoing traffic based on its rules.

---

## Types of Rules

### 1. Inbound Rules

- Control traffic **coming into** the EC2 instance.
- Example:

| Protocol | Port | Source   |
| -------- | ---- | -------- |
| SSH      | 22   | Your IP  |
| HTTP     | 80   | Anywhere |
| HTTPS    | 443  | Anywhere |

---

### 2. Outbound Rules

- Control traffic **leaving** the EC2 instance.
- By default, all outbound traffic is allowed.

---

## Common Ports

| Port | Service       |
| ---- | ------------- |
| 22   | SSH (Linux)   |
| 80   | HTTP          |
| 443  | HTTPS         |
| 3389 | RDP (Windows) |

---

## Security Group Characteristics

* Works at the **instance level**
* **Stateful** firewall
* Allows traffic only
* Can be attached to multiple EC2 instances
* Supports multiple rules

### What does "Stateful" mean?

If an inbound request is allowed, the response is automatically allowed. You don't need to create a separate outbound rule for the response.

---

## Example

A web server requires:

* SSH (22) for administration
* HTTP (80) for website access
* HTTPS (443) for secure website access

Inbound Rules:

| Port | Purpose |
| ---- | ------- |
| 22   | SSH     |
| 80   | HTTP    |
| 443  | HTTPS   |


---

## Key Points

* Security Groups act as virtual firewalls.
* They control inbound and outbound traffic.
* They are **stateful**.
* They are attached to EC2 instances.
* By default, all outbound traffic is allowed.
