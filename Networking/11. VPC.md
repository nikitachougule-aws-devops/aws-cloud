## Amazon VPC (Virtual Private Cloud)

- It is a private virtual network in AWS where you can launch and manage your AWS resources securely.
- It gives you complete control over your network, including IP addresses, subnets, routing, and security.

---

## VPC Components

A VPC typically includes:

* Subnets
* Route Tables
* Internet Gateway
* NAT Gateway
* Security Groups
* Network ACLs

---

## How a VPC Works

```text id="4c3q2v"
AWS Cloud
    │
    ▼
+----------------------+
|        VPC           |
|                      |
|  Public Subnet       |
|  Private Subnet      |
|                      |
+----------------------+
```

Resources such as EC2 instances, databases, and load balancers are launched inside a VPC.

---

## Public vs Private Subnet

| Public Subnet        | Private Subnet                           |
| -------------------- | ---------------------------------------- |
| Has internet access  | No direct internet access                |
| Used for web servers | Used for databases and internal services |

---

## Default VPC

Every AWS account comes with a **Default VPC** in each Region.

Features:

* Ready to use
* Includes public subnets
* Internet access is already configured

---

## Custom VPC

A **Custom VPC** is created by the user.

It allows you to:

* Choose your own IP address range (CIDR block)
* Create public and private subnets
* Configure routing and security

Custom VPCs are commonly used in production environments.

---

## Key Points

* VPC stands for **Virtual Private Cloud**.
* It is a private network inside AWS.
* Resources such as EC2 instances run inside a VPC.
* A VPC can contain public and private subnets.
* Every AWS account has a Default VPC.
