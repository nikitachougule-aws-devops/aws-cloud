## Internet Gateway (IGW)

- An Internet Gateway enables communication between a VPC and the internet.
- It is a VPC component that allows resources inside a **public subnet** to communicate with the internet.
- Without an Internet Gateway, EC2 instances cannot send or receive internet traffic.

Think of an Internet Gateway as the **door between your VPC and the Internet**.

---

## How Internet Gateway Works

```text
Internet
    │
    ▼
Internet Gateway
    │
    ▼
Public Subnet
    │
    ▼
EC2 Instance
```

---

## Requirements for Internet Access

For an EC2 instance to access the internet, it must have:

* An Internet Gateway attached to the VPC
* A route to the Internet Gateway in the Route Table
* A Public IP or Elastic IP
* A Security Group that allows the required traffic

---

## Public vs Private Subnet

| Public Subnet                 | Private Subnet                    |
| ----------------------------- | --------------------------------- |
| Connected to Internet Gateway | No direct Internet Gateway access |
| Can access the internet       | No direct internet access         |
| Used for web servers          | Used for databases                |

---

## Key Points

* Internet Gateway (IGW) connects a VPC to the internet.
* It is attached to a VPC.
* Public subnets use an Internet Gateway for internet access.
* Private subnets do not have direct internet access.
* An EC2 instance also needs a Public IP and proper routing to access the internet.
