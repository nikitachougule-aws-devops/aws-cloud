## NAT Gateway

- A NAT Gateway enables EC2 instances in private subnets to access the internet while preventing direct inbound connections.
- It is commonly used to allow private resources to download updates, install software, and access external services securely.
  
A NAT Gateway provides **outbound internet access only**.

---

## How NAT Gateway Works

```text
Internet
    ▲
    │
Internet Gateway
    ▲
    │
NAT Gateway (Public Subnet)
    ▲
    │
Private Subnet
    │
EC2 Instance
```

The EC2 instance can send requests to the internet, but the internet cannot directly connect to it.

---

## Where is a NAT Gateway Placed?

A NAT Gateway must be:

* Created in a **Public Subnet**
* Associated with an **Elastic IP**
* Connected to an **Internet Gateway**

Private subnets use a Route Table to send internet traffic to the NAT Gateway.

---

## NAT Gateway vs Internet Gateway

| NAT Gateway             | Internet Gateway               |
| ----------------------- | ------------------------------ |
| Used by private subnets | Used by public subnets         |
| Outbound internet only  | Inbound and outbound internet  |
| Requires an Elastic IP  | Does not require an Elastic IP |

---

## Example

Suppose you have:

* A web server in a **Public Subnet**
* A database server in a **Private Subnet**

The database server needs to download operating system updates.

Instead of making the database publicly accessible, it uses a NAT Gateway to access the internet securely.

---

## Key Points

* A NAT Gateway provides internet access to private subnets.
* It allows **outbound** internet traffic only.
* It must be deployed in a public subnet.
* It requires an Elastic IP.
* It works with an Internet Gateway.
