## AWS CLI (Command Line Interface)

- It is a tool that allows you to manage AWS services from the terminal or command prompt.
- Instead of using the AWS Management Console, you can run commands to create, update, and manage AWS resources.

---

## Why Use AWS CLI?

* Manage AWS services from the terminal
* Automate tasks using scripts
* Faster than using the AWS Console
* Useful for DevOps and Automation

---

## Install AWS CLI

Download and install AWS CLI from the official AWS website.

After installation, verify it:

```bash
aws --version
```

Example output:

```text
aws-cli/2.x.x
```

---

## Configure AWS CLI

Run the following command:

```bash
aws configure
```

You'll be prompted to enter:

```text
AWS Access Key ID:
AWS Secret Access Key:
Default region name:
Default output format:
```

Example:

```text
AWS Access Key ID: AKIA************
AWS Secret Access Key: ****************
Default region name: ap-south-1
Default output format: json
```

## Common AWS CLI Commands

### Check CLI Version

```bash
aws --version
```

### List S3 Buckets

```bash
aws s3 ls
```

### List EC2 Instances

```bash
aws ec2 describe-instances
```

### List IAM Users

```bash
aws iam list-users
```

### Copy File to S3

```bash
aws s3 cp image.jpg s3://my-bucket/
```

### Download File from S3

```bash
aws s3 cp s3://my-bucket/image.jpg .
```

---

## AWS CLI Command Format

General syntax:

```bash
aws <service> <operation> [options]
```

Example:

```bash
aws s3 ls
```

* `aws` → CLI command
* `s3` → AWS service
* `ls` → Operation

---

## Key Points

* AWS CLI is a command-line tool for managing AWS services.
* Configure it using `aws configure`.
* Uses IAM credentials for authentication.
* Supports automation and scripting.
* Commonly used by DevOps and Cloud Engineers.
