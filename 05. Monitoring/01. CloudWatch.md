## Amazon CloudWatch

- It is a monitoring and observability service that helps you monitor AWS resources and applications.
- It collects metrics, logs, and events so you can track performance and identify issues.
- CloudWatch tells you **how your AWS resources are performing**.

---

## CloudWatch Features

### 1. Metrics

Metrics are performance data collected from AWS resources.

Examples:

* CPU Utilization
* Network In/Out
* Disk Read/Write
* Memory Usage (with CloudWatch Agent)

---

### 2. Logs

CloudWatch Logs store log files from AWS services and applications.

Examples:

* EC2 application logs
* Lambda logs
* System logs

---

### 3. Alarms

CloudWatch Alarms monitor metrics and trigger actions when a condition is met.

Example:

* CPU Utilization > 80%
* Send an email notification
* Launch another EC2 instance using Auto Scaling

---

### 4. Events (EventBridge)

CloudWatch Events (now Amazon EventBridge) can trigger actions based on AWS events.

Example:

* Trigger a Lambda function when an EC2 instance stops.

## Common Use Cases

* Monitor EC2 CPU usage
* View application logs
* Receive alerts for high resource usage
* Trigger Auto Scaling
* Troubleshoot application issues

---


## Key Points

* CloudWatch is AWS's monitoring service.
* It collects **Metrics**, **Logs**, and **Events**.
* Alarms can send notifications or trigger automated actions.
* Commonly used with EC2, Lambda, RDS, and Auto Scaling.
