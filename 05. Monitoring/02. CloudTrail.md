## AWS CloudTrail

- It is a service that records and tracks API activity in your AWS account.
- It logs **who performed an action, what action was performed, when it happened, and from where**.

CloudTrail tells you **who did what in your AWS account**.

---

## Why Use CloudTrail?

* Track user activity
* Audit AWS account changes

---

## What Does CloudTrail Record?

CloudTrail records API calls made through:

* AWS Management Console
* AWS CLI
* AWS SDKs
* AWS APIs

---

## How CloudTrail Works

```text id="m9x7wa"
User / AWS CLI / SDK
         │
         ▼
     AWS API Call
         │
         ▼
     CloudTrail
         │
         ▼
     Event History / S3
```

---

## Event History

CloudTrail provides an **Event History** where you can view recent AWS account activity.

You can search events by:

* Event name
* User
* Service
* Time

---

## CloudTrail Logs

CloudTrail can automatically deliver logs to an **Amazon S3 bucket** for long-term storage and analysis.

You can also integrate CloudTrail with:

* CloudWatch Logs
* Amazon EventBridge


---

## CloudWatch vs CloudTrail

| CloudWatch                    | CloudTrail                         |
| ----------------------------- | ---------------------------------- |
| Monitors resource performance | Records AWS API activity           |
| Collects metrics and logs     | Collects account activity          |
| Creates alarms                | Supports auditing and compliance   |
| Example: CPU Utilization      | Example: User created an S3 bucket |

---

## Key Points

* CloudTrail records AWS API calls.
* It tracks **who**, **what**, **when**, and **from where**.
* Useful for auditing, security, and compliance.
* Logs can be stored in Amazon S3.
* Works with the AWS Console, CLI, SDKs, and APIs.
