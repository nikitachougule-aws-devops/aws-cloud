## Route Tables

- It is a set of rules that determines where network traffic from a subnet is directed.
- Every subnet in a VPC must be associated with a Route Table.

Think of a Route Table as a **GPS** that tells traffic where to go.

---

## How Route Tables Work

```text
EC2 Instance
      │
      ▼
Route Table
      │
      ├── Internet Gateway
      ├── NAT Gateway
      └── Local Route (VPC)
```

The Route Table checks the destination IP address and forwards traffic to the correct target.

---

## Default Route Table

- When you create a VPC, AWS automatically creates a **Default Route Table**.
- It contains a **Local Route**, allowing communication between subnets in the same VPC.

Example:

| Destination | Target |
| ----------- | ------ |
| 10.0.0.0/16 | Local  |

---

## Public Route Table

- A Public Route Table includes a route to the **Internet Gateway (IGW)**.
- Example:

| Destination | Target           |
| ----------- | ---------------- |
| 10.0.0.0/16 | Local            |
| 0.0.0.0/0   | Internet Gateway |

- This allows resources in a public subnet to access the internet.

---

## Private Route Table

- A Private Route Table sends internet-bound traffic to a **NAT Gateway**.
- Example:

| Destination | Target      |
| ----------- | ----------- |
| 10.0.0.0/16 | Local       |
| 0.0.0.0/0   | NAT Gateway |

- This allows instances in private subnets to access the internet without being directly reachable.

---

## Route Table Association

- A Route Table must be associated with one or more subnets.

```text
VPC
│
├── Public Subnet
│      │
│      └── Public Route Table
│
└── Private Subnet
       │
       └── Private Route Table
```

---

## Key Points

* A Route Table controls network traffic within a VPC.
* Every subnet must be associated with a Route Table.
* Public Route Tables use an Internet Gateway.
* Private Route Tables use a NAT Gateway.
* The Local Route enables communication within the VPC.
