## Amazon VPC Subnets

- It is a smaller network created inside a VPC.
- It divides a VPC into multiple sections, allowing you to organize and secure your AWS resources.
- A VPC can contain one or more subnets.

---

## Types of Subnets

### 1. Public Subnet

- A **Public Subnet** has a route to the Internet Gateway, allowing resources to access the internet.
- Common resources: Web Servers, Bastion Hosts, Load Balancers

---

### 2. Private Subnet

- A **Private Subnet** does not have a direct route to the Internet Gateway.
- Common resources: Databases, Application Servers, Internal Services
- Private subnets can access the internet using a **NAT Gateway**.

---

## How Subnets Work

```text
VPC (10.0.0.0/16)
│
├── Public Subnet (10.0.1.0/24)
│      └── EC2 Web Server
│
└── Private Subnet (10.0.2.0/24)
       └── RDS Database
```

---

## Subnet and Availability Zone

- A subnet belongs to **only one Availability Zone (AZ)**.
- To build a highly available application, create subnets in multiple AZs.

Example:

```text
Mumbai Region
│
├── AZ-a
│     └── Public Subnet
│
└── AZ-b
      └── Public Subnet
```

---

## Public vs Private Subnet

| Public Subnet         | Private Subnet                          |
| --------------------- | --------------------------------------- |
| Internet access       | No direct internet access               |
| Uses Internet Gateway | Uses NAT Gateway for outbound internet  |
| Hosts web servers     | Hosts databases and application servers |

---

## Key Points

* A subnet is a network inside a VPC.
* Every subnet belongs to one Availability Zone.
* Public subnets have internet access.
* Private subnets do not have direct internet access.
* Use multiple subnets for better security and availability.
