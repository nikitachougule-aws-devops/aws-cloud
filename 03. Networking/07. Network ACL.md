## Network ACL (Access Control List)

- It is a firewall that controls **inbound** and **outbound** traffic at the **subnet level**.
- It adds an extra layer of security for all resources within a subnet.
- A Network ACL protects the entire subnet, while a Security Group protects individual EC2 instances.

---

## How Network ACL Works

```text
Internet
    │
    ▼
Network ACL
    │
    ▼
Subnet
    │
    ▼
EC2 Instance
```

Every packet entering or leaving a subnet is checked against the Network ACL rules.

---

## Types of Rules

### 1. Inbound Rules

Control traffic entering the subnet.

### 2. Outbound Rules

Control traffic leaving the subnet.

Each rule has:

* Rule Number
* Protocol
* Port
* Source/Destination
* Allow or Deny action

---

## Security Group vs Network ACL

| Security Group            | Network ACL                   |
| ------------------------- | ----------------------------- |
| Works at instance level   | Works at subnet level         |
| Stateful                  | Stateless                     |
| Supports only Allow rules | Supports Allow and Deny rules |
| Applied to EC2 instances  | Applied to subnets            |

### What does "Stateless" mean?

If inbound traffic is allowed, you must also allow the corresponding outbound traffic. Responses are **not** automatically allowed.

---

## Example

Allow web traffic to a subnet:

| Rule       | Port | Action |
| ---------- | ---- | ------ |
| HTTP       | 80   | Allow  |
| HTTPS      | 443  | Allow  |
| All Others | -    | Deny   |

---

## Default Network ACL

Every VPC has a **Default Network ACL**.

* Allows all inbound traffic
* Allows all outbound traffic

You can create **Custom Network ACLs** with your own rules.

---

## Key Points

* Network ACL (NACL) protects an entire subnet.
* It controls inbound and outbound traffic.
* It is **stateless**.
* It supports both **Allow** and **Deny** rules.
* Every subnet must be associated with a Network ACL.
