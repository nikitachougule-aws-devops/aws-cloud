## Elastic IP (EIP)

- It is a **static public IPv4 address** provided by AWS.
- Unlike a normal public IP, an Elastic IP remains the same even if you stop and start your EC2 instance.
- Use an Elastic IP when your application needs a fixed public IP address.

---

## Public IP vs Elastic IP

| Public IP                                  | Elastic IP                      |
| ------------------------------------------ | ------------------------------- |
| Assigned automatically                     | Allocated manually              |
| Changes when the instance stops and starts | Remains the same until released |
| Temporary                                  | Static                          |

---

## How Elastic IP Works

```text
Internet
    │
    ▼
Elastic IP
    │
    ▼
EC2 Instance
```

If the EC2 instance fails, you can attach the same Elastic IP to another EC2 instance.

---

## How to Allocate an Elastic IP

1. Open the EC2 Dashboard.
2. Go to **Elastic IPs**.
3. Click **Allocate Elastic IP Address**.
4. Associate it with an EC2 instance.

---

## Pricing

* An Elastic IP associated with a running EC2 instance is generally free.
* AWS charges for unused (unassociated) Elastic IPs and in some other cases, encouraging efficient use of public IPv4 addresses.

Release Elastic IPs that you no longer need to avoid unnecessary charges.

---

## Key Points

* Elastic IP is a **static public IPv4 address**.
* It can be associated with an EC2 instance.
* It remains the same until you release it.
* It can be moved between EC2 instances.
* AWS may charge for unused Elastic IPs.
