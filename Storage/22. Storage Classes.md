## Amazon S3 Storage Classes

- **S3 Storage Classes** define how your data is stored and how much it costs.
- Choose a storage class based on **how often you access your data**.

---

## Types of S3 Storage Classes

| Storage Class                     | Best For                                                        |
| --------------------------------- | --------------------------------------------------------------- |
| **S3 Standard**                   | Frequently accessed data                                        |
| **S3 Intelligent-Tiering**        | Data with unknown or changing access patterns                   |
| **S3 Standard-IA**                | Infrequently accessed data                                      |
| **S3 One Zone-IA**                | Infrequently accessed data stored in a single Availability Zone |
| **S3 Glacier Instant Retrieval**  | Archived data that needs quick access                           |
| **S3 Glacier Flexible Retrieval** | Long-term archive                                               |
| **S3 Glacier Deep Archive**       | Rarely accessed data with the lowest storage cost               |

---

## 1. S3 Standard

* Frequently accessed data
* Low latency
* High availability
* Highest storage cost

**Examples:**

* Websites
* Mobile applications
* Frequently used files

---

## 2. S3 Intelligent-Tiering

* Automatically moves data between storage tiers
* Helps reduce storage costs
* No need to predict access patterns

**Examples:**

* User uploads
* Business documents

---

## 3. S3 Standard-IA (Infrequent Access)

* Lower storage cost than Standard
* Retrieval charges apply
* Suitable for data accessed occasionally

**Examples:**

* Backups
* Disaster recovery files

---

## 4. S3 One Zone-IA

* Stores data in a single Availability Zone
* Lower cost than Standard-IA
* Not suitable for critical data

**Examples:**

* Temporary backups
* Re-creatable data

---

## 5. S3 Glacier Instant Retrieval

* Low-cost archive storage
* Millisecond retrieval
* For data rarely accessed but needed immediately

**Examples:**

* Medical records
* Archived images

---

## 6. S3 Glacier Flexible Retrieval

* Very low storage cost
* Retrieval takes minutes to hours

**Examples:**

* Long-term backups
* Archived business data

---

## 7. S3 Glacier Deep Archive

* Lowest storage cost
* Retrieval takes several hours
* Designed for long-term retention

**Examples:**

* Legal documents
* Financial records
* Compliance archives

---

## Storage Class Comparison

| Storage Class              | Access Frequency  | Retrieval Speed  | Cost     |
| -------------------------- | ----------------- | ---------------- | -------- |
| Standard                   | Frequent          | Milliseconds     | High     |
| Intelligent-Tiering        | Unknown           | Milliseconds     | Medium   |
| Standard-IA                | Rare              | Milliseconds     | Lower    |
| One Zone-IA                | Rare              | Milliseconds     | Low      |
| Glacier Instant Retrieval  | Very Rare         | Milliseconds     | Lower    |
| Glacier Flexible Retrieval | Archive           | Minutes to Hours | Very Low |
| Glacier Deep Archive       | Long-term Archive | Hours            | Lowest   |

---

## Which Storage Class Should You Choose?

| Use Case                         | Recommended Storage Class  |
| -------------------------------- | -------------------------- |
| Website images                   | S3 Standard                |
| User uploads                     | Intelligent-Tiering        |
| Backups                          | Standard-IA                |
| Temporary backups                | One Zone-IA                |
| Archived files with quick access | Glacier Instant Retrieval  |
| Long-term backups                | Glacier Flexible Retrieval |
| Compliance and legal records     | Glacier Deep Archive       |

---

## Best Practices

* Use **Standard** for frequently accessed data.
* Use **Intelligent-Tiering** when access patterns are unpredictable.
* Use **Glacier** storage classes for long-term archives.
* Configure **Lifecycle Rules** to automatically move objects between storage classes.
