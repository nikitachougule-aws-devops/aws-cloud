## AWS Global Infrastructure

- AWS Global Infrastructure is the worldwide network of data centers that AWS uses to provide cloud services. 
- It is designed for **high availability, low latency, and fault tolerance**.

Main components:

* Regions
* Availability Zones (AZs)
* Edge Locations

---

## AWS Region

- It is a physical geographic location where AWS has multiple data centers.

Examples:

* Mumbai
* Singapore
* London
* Frankfurt
* Ohio

Choose a Region based on:

* User location
* Pricing
* Compliance requirements
* Service availability

> Example: If most users are in India, choose the **Mumbai Region**.

---

## Availability Zone (AZ)

- It is one or more isolated data centers within a Region.
- A Region contains multiple AZs.
- Using multiple AZs improves application availability.

Example:

```text
Mumbai Region
├── ap-south-1a
├── ap-south-1b
└── ap-south-1c
```


---

## Region vs Availability Zone

| Region                     | Availability Zone          |
| -------------------------- | -------------------------- |
| Geographic area            | Data center(s)             |
| Contains multiple AZs      | Belongs to one Region      |
| Used for disaster recovery | Used for high availability |

---

## Edge Locations

- Edge Locations are used by AWS services like **CloudFront** to deliver content faster.
- They cache data closer to users, reducing latency.

---

## Key Points

* **Region** = Geographic location.
* **Availability Zone** = One or more isolated data centers inside a Region.
* **Edge Location** = Caches content closer to users.
* Multiple AZs improve availability.
* Multiple Regions improve disaster recovery.
