## AWS

- It is the world's largest cloud computing platform.
- It provides hundreds of on-demand services for computing, storage, databases, networking, security, analytics, machine learning, and more.
- Instead of purchasing and maintaining physical servers, organizations can rent IT resources from AWS over the internet and pay only for what they use.
- AWS was launched by Amazon in **2006** and has since become the leading cloud platform used by startups, enterprises, governments, and educational institutions.

---

## Cloud Computing

Cloud computing is the delivery of computing resources over the internet instead of using local computers or on-premises data centers.

These resources include:

* Virtual servers
* Storage
* Databases
* Networking
* Security
* Artificial Intelligence (AI)
* Machine Learning (ML)
* Monitoring
* DevOps tools

Instead of buying hardware, installing operating systems, and managing infrastructure manually, users can provision resources within minutes using AWS.

---

## Why Use AWS?

AWS helps organizations:

* Reduce infrastructure costs
* Deploy applications quickly
* Scale automatically
* Improve availability
* Increase security
* Expand globally with minimal effort

Whether you're building a small website or a global application, AWS provides services that fit almost every workload.

## Benefits of AWS

### 1. Pay-as-You-Go Pricing

- You pay only for the resources you consume.
- Example: Run an EC2 instance for 2 hours. Pay only for those 2 hours.
- No long-term hardware investment is required.

---

### 2. Scalability

- AWS resources can scale based on demand.
- Example: A shopping website receives heavy traffic during a festival sale.
- Instead of purchasing additional physical servers, AWS can automatically launch more instances and remove them when traffic decreases.

---

### 3. High Availability

- AWS provides infrastructure across multiple geographic locations.
- Applications can be deployed across different Availability Zones so that if one data center fails, another continues serving users.

---

### 4. Global Reach

- AWS operates data centers around the world.
- Applications can be deployed close to users, reducing latency and improving performance.

---

### 5. Security

- AWS provides enterprise-grade security features such as: IAM, Encryption, Firewalls, Security Groups, Network ACLs, Compliance certifications
- Customers remain responsible for securing their own workloads according to the AWS Shared Responsibility Model.

---

### 6. Reliability

- AWS services are designed with redundancy and fault tolerance.
- Data can be replicated across multiple locations to minimize downtime and reduce the risk of data loss.

---

### 7. Elasticity

- Elasticity allows AWS resources to automatically increase or decrease based on workload.
- Example: A video streaming platform experiences high traffic in the evening.
- AWS automatically adds resources during peak hours and removes them afterward.

## Cloud Service Models

Cloud services are commonly categorized into three models.

### Infrastructure as a Service (IaaS)

- AWS provides virtual infrastructure while the customer manages the operating system, applications, and data.
- Examples: Amazon EC2, Amazon EBS, Amazon VPC
- Use case: Hosting web applications with full control over the operating system.

---

### Platform as a Service (PaaS)

- AWS manages the underlying infrastructure and runtime while developers focus on building applications.
- Examples: AWS Elastic Beanstalk, AWS Lambda
- Use case: Developing and deploying applications without managing servers.

---

### Software as a Service (SaaS)

- Complete software applications delivered over the internet.
- Examples: Gmail, Microsoft 365, Salesforce
- AWS provides infrastructure that many SaaS companies use to host their applications.

---

## Types of Cloud Deployment

### Public Cloud

- Infrastructure is owned and managed by a cloud provider such as AWS.
- Resources are shared among multiple customers but remain logically isolated.
- Example: Hosting a personal website on AWS.

---

### Private Cloud

- Infrastructure is dedicated to a single organization.
- Often used by banks, government agencies, and organizations with strict compliance requirements.

---

### Hybrid Cloud

- Combines on-premises infrastructure with cloud services.
- Example: An organization stores sensitive data on-premises while hosting customer-facing applications on AWS.

## AWS Shared Responsibility Model

Security in AWS is a shared responsibility between AWS and the customer.

### AWS is responsible for ("Security **of** the Cloud")

* Physical data centers
* Networking hardware
* Storage hardware
* Global infrastructure
* Hypervisor
* Managed service infrastructure

### Customers are responsible for ("Security **in** the Cloud")

* IAM users and permissions
* Application security
* Data encryption
* Operating system updates (for EC2)
* Network configurations
* Security Groups
* Customer data

A simple way to remember this is:

> AWS secures the infrastructure, while customers secure what they build on that infrastructure.

## Key Terms

**Cloud Computing**
Delivery of computing resources over the internet.

**AWS Region**
A geographic area containing multiple Availability Zones.

**Availability Zone (AZ)**
One or more isolated data centers within a Region.

**EC2**
Virtual server service.

**S3**
Object storage service.

**IAM**
Identity and Access Management service.

**VPC**
Virtual private network for AWS resources.

**Lambda**
Serverless compute service.

**RDS**
Managed relational database service.

## Example

Imagine you are launching an e-commerce website.

Without AWS, you would need to:

* Purchase servers
* Rent data center space
* Install networking equipment
* Configure storage
* Manage hardware failures
* Plan for traffic spikes

With AWS, you can:

* Launch EC2 instances in minutes.
* Store product images in S3.
* Use RDS for the database.
* Distribute traffic with an Elastic Load Balancer.
* Scale automatically using Auto Scaling.
* Monitor the application with CloudWatch.

This lets you focus on building the application instead of managing infrastructure.
