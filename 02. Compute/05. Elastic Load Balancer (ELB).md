## Elastic Load Balancer (ELB)

- It is an AWS service that automatically distributes incoming traffic across multiple EC2 instances.
- It helps improve **availability, scalability, and fault tolerance**.
-  Instead of sending all traffic to one server, ELB distributes it across multiple servers.

---

## How ELB Works

```text
           Users
             │
             ▼
      Elastic Load Balancer
         ┌──────┴──────┐
         ▼             ▼
      EC2-1         EC2-2
```

If one EC2 instance becomes unhealthy, ELB sends traffic only to the healthy instance.

---

## Types of Load Balancers

### 1. Application Load Balancer (ALB)

* Works at **Layer 7 (HTTP/HTTPS)**
* Best for web applications
* Supports path-based and host-based routing

Example:

* `/api` → EC2 Instance A
* `/images` → EC2 Instance B

---

### 2. Network Load Balancer (NLB)

* Works at **Layer 4 (TCP/UDP)**
* Handles millions of requests
* Best for high-performance applications

---

### 3. Gateway Load Balancer (GWLB)

* Used for deploying and scaling virtual network appliances
* Commonly used with firewalls and security appliances

---

## Health Checks

ELB continuously checks whether EC2 instances are healthy.

* **Healthy instance** → Receives traffic
* **Unhealthy instance** → Stops receiving traffic

This improves application reliability.

---

## ELB with Auto Scaling

ELB and Auto Scaling are commonly used together.

```text
            Users
              │
              ▼
             ELB
       ┌──────┴──────┐
       ▼             ▼
    EC2-1         EC2-2
       │             │
       └──────┬──────┘
              ▼
     Auto Scaling Group
```

* ELB distributes traffic.
* Auto Scaling adds or removes EC2 instances based on demand.

---

## Key Points

* ELB stands for **Elastic Load Balancer**.
* It distributes traffic across multiple EC2 instances.
* It performs health checks automatically.
* Works seamlessly with Auto Scaling.
* AWS provides **Application**, **Network**, and **Gateway** Load Balancers.
