## Amazon EC2 (Elastic Compute Cloud)

- It is a service that provides virtual servers in the cloud.
- You can launch, stop, start, and terminate servers whenever needed without buying physical hardware.
- EC2 is one of the most widely used AWS services.

---

## Why Use EC2?

* Host websites and applications
* Run databases
* Deploy APIs
* Run Docker containers
* Create development and testing environments

---

## EC2 Features

* Virtual servers on demand
* Multiple operating systems (Linux, Windows)
* Easy scaling
* Secure with Security Groups
* Pay only for what you use

---

## EC2 Instance

An **EC2 Instance** is a virtual machine running in AWS.

When creating an instance, you choose:

* Amazon Machine Image (AMI)
* Instance Type
* Storage
* Security Group
* Key Pair

---

## EC2 Launch Process

```text
Choose AMI
      ↓
Choose Instance Type
      ↓
Configure Instance
      ↓
Add Storage
      ↓
Configure Security Group
      ↓
Create/Select Key Pair
      ↓
Launch EC2 Instance
```

---

## Instance Types

AWS provides different instance types for different workloads.

| Type                     | Best For           |
| ------------------------ | ------------------ |
| General Purpose (T, M)   | Web applications   |
| Compute Optimized (C)    | High CPU workloads |
| Memory Optimized (R)     | Databases          |
| Storage Optimized (I, D) | High-speed storage |

---

## Public IP vs Private IP

| Public IP                    | Private IP                      |
| ---------------------------- | ------------------------------- |
| Accessible from the internet | Accessible only within the VPC  |
| Used for remote access       | Used for internal communication |

---

## Connect to an EC2 Instance

### Linux

Use SSH:

```bash
ssh -i my-key.pem ec2-user@<public-ip>
```

### Windows

Use **Remote Desktop Protocol (RDP)** with the Administrator password.

---

## EC2 States

| State      | Description                     |
| ---------- | ------------------------------- |
| Pending    | Instance is launching           |
| Running    | Instance is running             |
| Stopping   | Instance is shutting down       |
| Stopped    | Instance is powered off         |
| Terminated | Instance is permanently deleted |

> A terminated instance cannot be restarted.

---

## Stop vs Terminate

| Stop                             | Terminate             |
| -------------------------------- | --------------------- |
| Can start again                  | Cannot be recovered   |
| Data on EBS remains              | Instance is deleted   |
| No compute charges while stopped | Resources are removed |

---

## EC2 Pricing Options

* **On-Demand** – Pay only for usage.
* **Reserved Instances** – Lower cost with a long-term commitment.
* **Spot Instances** – Low-cost instances that AWS can interrupt.
* **Savings Plans** – Reduced pricing based on committed usage.

---

## Key Points

* EC2 provides virtual servers in AWS.
* An EC2 Instance is a virtual machine.
* Choose an AMI, instance type, storage, and security settings before launching.
* Use SSH for Linux and RDP for Windows.
* EC2 supports multiple pricing options.
