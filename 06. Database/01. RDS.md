# Amazon RDS (Relational Database Service)

- It is a fully managed relational database service.
- It makes it easy to create, operate, and scale relational databases without managing the underlying servers.

RDS lets you focus on your application while AWS manages the database.

---

## Supported Database Engines

* MySQL
* PostgreSQL
* MariaDB
* Oracle
* Microsoft SQL Server

---

## How RDS Works

```text id="ljv3rk"
Application
      │
      ▼
   Amazon RDS
      │
      ▼
Database Engine
(MySQL, PostgreSQL, etc.)
```

Applications connect to the RDS database using its **endpoint**.

---

## Multi-AZ

**Multi-AZ** creates a standby database in another Availability Zone.

Benefits:

* High availability
* Automatic failover
* Better reliability

---

## Read Replica

A **Read Replica** is a copy of the primary database used for read operations.

Benefits:

* Improves read performance
* Reduces load on the primary database


---

## RDS vs EC2 Database

| Amazon RDS         | Database on EC2 |
| ------------------ | --------------- |
| Fully managed      | Self-managed    |
| Automatic backups  | Manual backups  |
| Automatic patching | Manual patching |
| Easy scaling       | Manual scaling  |

---

## Key Points

* RDS is a managed relational database service.
* Supports MySQL, PostgreSQL, MariaDB, Oracle, and SQL Server.
* Multi-AZ provides high availability.
* Read Replicas improve read performance.
* AWS handles backups, patching, and maintenance.
