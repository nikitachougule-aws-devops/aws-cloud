## Amazon EFS (Elastic File System)

- It is a fully managed file storage service for AWS.
- It provides a **shared file system** that can be accessed by multiple EC2 instances at the same time.

Think of EFS as a **shared network drive** for EC2 instances.

---

## How EFS Works

```text id="4fj9yq"
          Amazon EFS
         /     |     \
        /      |      \
     EC2-1   EC2-2   EC2-3
```

All EC2 instances can read and write to the same file system.

---

## EFS Features

* Fully managed by AWS
* Shared storage
* Automatically scales as data grows
* High availability
* Supports multiple Availability Zones

---

## EFS vs EBS

| EFS                    | EBS                                       |
| ---------------------- | ----------------------------------------- |
| Shared storage         | Storage for one EC2 instance (by default) |
| File storage           | Block storage                             |
| Auto scales            | Fixed size (can be resized)               |
| Multiple EC2 instances | Single EC2 instance                       |

---

## Common Use Cases

* Web applications
* Content management systems (CMS)
* Shared application files
* Machine learning workloads
* Big data analytics

---

## Key Points

* EFS stands for **Elastic File System**.
* It is a shared file storage service.
* Multiple EC2 instances can access the same file system.
* It automatically scales as storage needs grow.
* Mainly used with Linux-based workloads.
